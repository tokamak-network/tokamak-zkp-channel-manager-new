# UI/UX Flow Documentation

이 문서는 UI/UX 리팩토링 시 화면 간 흐름과 사용자 인터랙션을 기록합니다.

---

## 📱 페이지 플로우 개요

### 메인 플로우

```
홈 → 채널 생성 → 입금 → 상태 초기화 → 거래 실행 → 증명 제출 → 출금
```

---

## 🏠 페이지별 상세 플로우

### 1. 홈 페이지 (`/`)

**화면 구성:**

- 상단: 헤더 (로고, 네비게이션, 계정 정보)
- 중앙: 채널 목록 카드들
- 각 카드에는: 채널 ID, 상태, 잔액 표시

**사용자 액션:**

- **"새 채널 생성" 버튼 클릭** → `/create-channel` 페이지로 이동
- **채널 카드 클릭** → 해당 채널 상세 페이지로 이동
- **"상태 탐색기" 메뉴 클릭** → `/state-explorer` 페이지로 이동

---

### 2. 채널 생성 페이지 (`/create-channel`)

**화면 구성:**

- 상단: "채널 생성" 제목
- 중앙: 채널 생성 폼
  - 참여자 주소 입력 필드
  - 초기 입금 금액 입력 필드
  - "생성" 버튼

**사용자 액션:**

- **"생성" 버튼 클릭** → 트랜잭션 서명 → 성공 시 `/deposit` 페이지로 이동
- **"취소" 버튼 클릭** → `/` (홈) 페이지로 이동

---

### 3. 입금 페이지 (`/deposit`)

**화면 구성:**

- 상단: "토큰 입금" 제목
- 중앙: 입금 폼
  - 채널 선택 드롭다운
  - 토큰 종류 선택
  - 입금 금액 입력 필드
  - "입금" 버튼

**사용자 액션:**

- **"입금" 버튼 클릭** → 토큰 승인 (필요시) → 입금 트랜잭션 서명 → 성공 시 `/initialize-state` 페이지로 이동
- **"다음 단계로" 버튼 클릭** → `/initialize-state` 페이지로 이동

---

### 4. 상태 초기화 페이지 (`/initialize-state`)

**화면 구성:**

- 상단: "상태 초기화" 제목
- 중앙: 초기 상태 설정 폼
  - 초기 잔액 정보 표시
  - 참여자 정보 표시
  - "초기화" 버튼

**사용자 액션:**

- **"초기화" 버튼 클릭** → 초기화 트랜잭션 서명 → 성공 시 `/state-explorer` 페이지로 이동
- **"상태 탐색기로" 버튼 클릭** → `/state-explorer` 페이지로 이동

---

### 5. 상태 탐색기 페이지 (`/state-explorer`)

**화면 구성:**

- 좌측: 사이드바 (채널 목록)
- 중앙: 상태 트리 뷰
  - 현재 상태 스냅샷
  - 거래 히스토리
  - "새 거래 생성" 버튼

**사용자 액션:**

- **"새 거래 생성" 버튼 클릭** → 거래 생성 모달 열림
- **거래 항목 클릭** → 거래 상세 정보 표시
- **"증명 제출" 버튼 클릭** → `/submit-proof` 페이지로 이동

---

### 6. 증명 제출 페이지 (`/submit-proof`)

**화면 구성:**

- 상단: "증명 제출" 제목
- 중앙: 증명 제출 폼
  - 채널 선택
  - 증명 데이터 표시
  - "제출" 버튼

**사용자 액션:**

- **"제출" 버튼 클릭** → 증명 검증 → 제출 트랜잭션 서명 → 성공 시 `/` (홈) 페이지로 이동
- **"취소" 버튼 클릭** → `/state-explorer` 페이지로 이동

---

## 🔄 모달 및 팝업 플로우

### 거래 생성 모달

**트리거:** 상태 탐색기에서 "새 거래 생성" 버튼 클릭

**화면 구성:**

- 거래 타입 선택 (전송, 교환 등)
- 수신자 주소 입력
- 금액 입력
- "생성" 버튼

**사용자 액션:**

- **"생성" 버튼 클릭** → 거래 생성 → 모달 닫힘 → 상태 탐색기 새로고침
- **"취소" 버튼 클릭** → 모달 닫힘

---

### 거래 번들 모달

**트리거:** 상태 탐색기에서 거래 번들 항목 클릭

**화면 구성:**

- 거래 번들 상세 정보
- 포함된 거래 목록
- "실행" 버튼

**사용자 액션:**

- **"실행" 버튼 클릭** → 거래 실행 → 모달 닫힘
- **"취소" 버튼 클릭** → 모달 닫힘

---

## 📝 참고사항

### 네비게이션 패턴

- 모든 페이지는 상단 헤더에 네비게이션 메뉴 포함
- 사이드바는 상태 탐색기에서만 표시
- 브레드크럼은 복잡한 플로우에서만 사용

### 상태 표시

- 로딩 상태: 각 페이지에서 로딩 스피너 표시
- 에러 상태: 에러 메시지 토스트 표시
- 성공 상태: 성공 메시지 토스트 표시 후 다음 페이지로 이동
